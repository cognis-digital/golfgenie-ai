Accessibility Statement

Last Updated: January 1, 2026

Our Commitment to Accessibility

Myrtle Beach Golf Tourism App is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards to ensure we provide equal access to all of our users.

Accessibility Standards

We strive to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards. These guidelines explain how to make web content more accessible for people with disabilities and user-friendly for everyone.

Current Accessibility Features

  • Keyboard Navigation: Full keyboard accessibility for all interactive elements
  • Screen Reader Support: Compatible with popular screen readers like NVDA, JAWS, and VoiceOver
  • High Contrast: Sufficient color contrast ratios for text and background elements
  • Responsive Design: Mobile-friendly design that works across all devices
  • Alternative Text: Descriptive alt text for all images and graphics
  • Focus Indicators: Clear visual focus indicators for keyboard navigation
  • Semantic HTML: Proper heading structure and semantic markup
  • Form Labels: Clear labels and instructions for all form fields
  • Error Messages: Clear, descriptive error messages and validation
  • Skip Links: Skip navigation links for efficient page navigation

Keyboard Navigation

Our application supports full keyboard navigation. Here are the primary keyboard shortcuts:

Tab: Navigate forward through interactive elements
Shift + Tab: Navigate backward through interactive elements
Enter/Space: Activate buttons and links
Escape: Close modals and dropdown menus
Arrow Keys: Navigate within menus and lists
Home/End: Jump to beginning/end of lists

Screen Reader Support

Our application is designed to work with screen readers and includes:

  • Proper ARIA labels and descriptions
  • Live regions for dynamic content updates
  • Descriptive page titles and headings
  • Clear navigation landmarks
  • Form field associations and error announcements

Visual Accessibility

We have implemented several features to support users with visual impairments:

  • High contrast color schemes meeting WCAG AA standards
  • Scalable text that can be enlarged up to 200% without loss of functionality
  • Clear, readable fonts with appropriate sizing
  • Visual focus indicators for all interactive elements
  • Icons accompanied by text labels

Motor Accessibility

For users with motor impairments, we provide:

  • Large touch targets (minimum 44px) for mobile devices
  • Sufficient spacing between interactive elements
  • No time-sensitive actions that cannot be extended
  • Alternative input methods for complex interactions

Cognitive Accessibility

To support users with cognitive disabilities:

  • Clear, simple language throughout the application
  • Consistent navigation and layout patterns
  • Clear error messages with suggestions for correction
  • Progress indicators for multi-step processes
  • Option to review and confirm important actions

Third-Party Content

While we strive to ensure accessibility across our entire platform, some third-party content (such as embedded maps, booking widgets, or external links) may not fully conform to our accessibility standards. We work with our partners to improve accessibility where possible.

Ongoing Improvements

We are continuously working to improve the accessibility of our application. Our efforts include:

  • Regular accessibility audits and testing
  • User testing with people with disabilities
  • Staff training on accessibility best practices
  • Incorporating accessibility into our development process
  • Staying current with accessibility standards and guidelines

Known Issues

We are aware of the following accessibility issues and are working to address them:

  • Some third-party map features may have limited keyboard accessibility
  • Certain dynamic content updates may not be immediately announced to screen readers
  • Some complex data tables may need improved navigation
